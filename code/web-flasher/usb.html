Badge Flasher — Help

Connect — Opens a Web Serial connection to the RP2040 running MicroPython. Once connected the button toggles to "Disconnect." If the device is unplugged the UI will update automatically.

Check for Updates — Downloads the latest firmware ZIP from the GitHub repo and stages it locally. It does not automatically write to the badge; you'll need to press "Upload Firmware" to install.

Choose / Load Firmware — "Choose firmware" opens a file picker; "Load Firmware" parses a local firmware ZIP (.zip) or single file (.uf2/.bin) into the web app's staging area. Loading does NOT modify the badge; it simply prepares the package for upload.

Upload Firmware — Sends files from the loaded firmware package to the badge via Web Serial. It creates folders if required and uploads files; the /VERSION file (if present) is uploaded last, to ensure devices apply updates correctly. Upload does NOT perform a wipe or factory reset — it will overwrite files with the same path but does not delete unrelated files.

Save Configuration — Writes the values in the "Name", "Slack handle" and "Pronouns" fields to /config.json on the badge. This operation does not disconnect the badge. The uploaded configuration will be used by the badge software where appropriate.

Load Configuration — Downloads /config.json from the badge (if it exists) and populates the form fields. It doesn't change other files.

Wipe Badge — Attempts to delete files from the device file system. This is a destructive operation; use with caution. It does NOT re-flash the bootloader or change the device's MicroPython firmware mode.

Limits / Notes — UF2-style flashing (device appears as a mass-storage device) cannot be performed via Web Serial. Upload via Web Serial writes files while the MicroPython firmware is running. If you need to re-flash the UF2, use the device USB boot mode and copy the UF2 file using the OS file manager.
